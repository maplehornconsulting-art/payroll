# Canada – Payroll CRA Auto-Update Connector (`l10n_ca_hr_payroll_cra_connector`)

> **Pricing:** This connector module is **free**. It requires the paid base module **Canada – Payroll** (`l10n_ca_hr_payroll_except_QC`) to function.

> **⚠ Disclaimer:** This module is **not** an official product of the Canada Revenue Agency (CRA)
> or the Government of Canada. Values are derived from publicly available CRA publications and are
> provided for convenience only. All payroll calculations **must be reviewed and approved by a
> qualified Canadian payroll professional** before being applied to live payroll. No warranty of any
> kind is provided. Use at your own risk.

---

## What this module does

`l10n_ca_hr_payroll_cra_connector` is a companion Odoo module for
[`l10n_ca_hr_payroll_except_QC`](https://github.com/maplehornconsulting-art/payroll/tree/main/l10n_ca_hr_payroll_except_QC).
It consumes the free, machine-readable CRA payroll tax feed published by MapleHorn Consulting via
GitHub Pages and writes approved updates into the `hr.rule.parameter` records that drive payroll
calculations.

**Default behaviour: human-approval gate is always on.**

A cron job fetches the latest feed daily. If a new version is detected it creates a draft
*CRA Tax Update* record and notifies the HR Payroll Manager group via email + Odoo activity.
The payroll manager reviews the proposed changes line-by-line and clicks **Apply** to write values
into `hr.rule.parameter`. Nothing is changed automatically without an explicit approval click.

---

## Human-approval workflow

1. **Fetch (automatic, daily):** The cron job calls
   `https://maplehornconsulting-art.github.io/payroll/v1/ca/latest.json`, verifies the built-in
   SHA-256 checksum, and creates a `draft` *CRA Tax Update* if the feed has changed.
2. **Review (manual):** The HR Payroll Manager receives an email notification and an Odoo activity.
   Open the record in **Payroll → Configuration → CRA Tax Updates**, review each line (old value,
   new value, mapped parameter), deselect any lines you do not want to apply.
3. **Apply (manual):** Click **Apply…**, confirm the disclaimer in the wizard, and click **Apply**.
   The module writes a new dated `hr.rule.parameter.value` row for each selected line.

## Configuration

Navigate to **Payroll → Configuration → Settings** → *Canadian Payroll – CRA Auto-Update* section.

| Setting | Default | Description |
|---|---|---|
| **Feed URL** | `https://maplehornconsulting-art.github.io/payroll/v1/ca/latest.json` | Override to point at a custom feed mirror. |
| **GPG Fingerprint** | *(empty)* | Optional: fingerprint of the GPG key used to sign the feed. Verification is a stub — see TODO in `cra_feed_client.py`. |
| **Auto-Apply** | **Off** | ⚠ When enabled, the cron will apply updates automatically if the signature verifies AND every line maps cleanly. **Not recommended for production.** Always have a payroll professional review changes. |

### Manual trigger

To fetch the feed immediately without waiting for the cron:

1. Go to **Payroll → Configuration → CRA Tax Updates**.
2. From any existing update record, click **Refetch Now**, or
3. Run the cron manually via **Settings → Technical → Scheduled Actions → CRA Tax Update Check**.

---

## How to roll back an applied update

1. Open the update record that was applied before the one you want to roll back to
   (**Payroll → Configuration → CRA Tax Updates**, filter by *Applied*).
2. Click **Apply…** again on the older update. This creates new `hr.rule.parameter.value` rows
   with the older effective date, which Odoo will use for payslips computed on or after that date.

## Upstream feed source

The feed is generated by a GitHub Actions workflow in this same repository:

- Feed URL: `https://maplehornconsulting-art.github.io/payroll/v1/ca/latest.json`
- Source code: `https://github.com/maplehornconsulting-art/payroll/tree/main/cra_feed`

## Security & compliance notes

- This module makes **outbound HTTP requests** to the feed URL. Ensure your Odoo server has
  outbound internet access to `maplehornconsulting-art.github.io`.
- The checksum verification (`checksum_sha256`) is always performed before any record is created.
- Optional GPG signature verification can be enabled by setting a GPG fingerprint in Settings.
  The actual GPG verification code is a **stub** in this release — see the `TODO` comment in
  `models/cra_feed_client.py`. Pull requests welcome.
- All user-facing actions are gated by `hr_payroll.group_hr_payroll_manager`.
